Below is **everything that exists in your HTML/React code** for the **Advanced Vehicle Dataset Annotation Tool**, grouped exactly into the buckets you asked for (Basic / Mechanical / Attributes / Category+Subcategory / Licence Plate), plus the other schema bits that are clearly part of the tool.

---

## 1) Core annotation schema (what the JSON contains)

### A) Top-level JSON structure

* `image`

  * `filename` (string)
  * `width` (number) — default 1920
  * `height` (number) — default 1080
  * `timestamp` (ISO string)
* `objects` (array of annotation objects)

### B) Every annotation object has (base fields)

* `id` (number; `Date.now() + Math.random()`)
* `type` (string) — depends on annotation type
* `bbox` (array `[x1, y1, x2, y2]` integers)

### C) Annotation *types* supported by the tool (drawing / labels)

* `vehicle`
* `license_plate`
* `wheel` (internally converted to either `singleWheel` or `doubleWheel`)
* `attachment`

---

## 2) Vehicle annotation fields (this is the “advanced vehicle dataset” part)

When `type === "vehicle"`, the tool writes these extra blocks:

### A) Basic (vehicle identity + context)

**Vehicle-level fields**

* `view_pose` (string)
* `direction` (string)

**`vehicle_info` object**

* `category` (string)
* `subcategory` (string)
* `country` (string)
* `brand` (string)
* `model` (string)
* `color` (string)
* `operator` (string)
* `number_of_seats` (number)

### B) Mechanical (vehicle physical/mechanical cues)

**`mechanical` object**

* `number_of_wheels_visible` (number)
* `number_of_axles_inferred` (number)
* `number_of_axles_raised` (number)
* `truck_trailer_labels_visible` (boolean)
* `cargo_present` (boolean)

### C) Attributes (functional/service/energy flags)

**`attributes` object**

* `is_taxi` (boolean)
* `is_bus` (boolean)
* `bus_type` (string)
* `is_emergency_vehicle` (boolean)
* `is_electric` (boolean)

### D) Licence plate information (vehicle-linked block)

**`license_plate` object** *(only created if `license_plate_visible` is true; otherwise `null`)*

* `bbox` (array `[0,0,0,0]` initially; comment says it will be updated when plate annotation is added)
* `text` (string)  ← from `license_plate_text`
* `country` (string) ← from `license_plate_country`
* `format` (string) ← from `license_plate_format`
* `blurred` (boolean)
* `visible` (boolean)

### E) Chain-of-Thought / notes (saved to JSON)

* `CoT` (array of strings)

  * Comes from `vehicleForm.chainOfThought` after filtering out empty lines.

### F) Relationship holders (present but not actively populated in the snippet)

These are always included on vehicle creation:

* `wheels` (array)
* `axles` (array)
* `attachments` (array)

---

## 3) Wheel annotations (what gets stored)

When drawing a wheel:

* The tool **overrides `type`** to one of:

  * `singleWheel`
  * `doubleWheel`
* Also stores:

  * `parent_id: null`

(You *do* also have UI logic referring to `obj.wheel_type` in places, but the created wheel object in this snippet sets `type` to `singleWheel/doubleWheel` rather than storing a stable `wheel_type` field.)

---

## 4) Licence plate annotations (as their own bbox object)

If you draw a plate as an annotation type (not the vehicle-linked licence_plate block), it becomes an object with:

* `id`
* `type: "license_plate"`
* `bbox`
* `parent_id: null`
* `attributes: {}`

---

## 5) Attachment annotations (trailer/attachment bbox objects)

If you draw an attachment:

* `id`
* `type: "attachment"`
* `bbox`
* `parent_id: null`
* `attributes: {}`
